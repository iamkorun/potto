See it in action

Run potto in any project — it finds your env files automatically.

~/my-project
$ potto
-> Checking .env against .env.example
FAIL 2 key(s) in .env are MISSING from .env.example (teammates can't run the app)
  - STRIPE_SECRET_KEY
  - REDIS_URL
WARN 1 key(s) in .env.example are MISSING from .env (you may need to set these)
  - FEATURE_FLAG_NEW_UI
Run `potto sync` to fix .env.example automatically.
$ potto sync
-> Adding 2 key(s) to .env.example

$ potto
OK All 12 key(s) are in sync.
Features

Small tool, big impact

Built for the one second before a teammate messages you "why is the app crashing?"

🔍
Zero Config
Auto-discovers .env and .env.example in the current directory. Just run potto.
🎨
Color-Coded Output
FAIL in red, WARN in yellow, OK in green. See problems at a glance without parsing walls of text.
⚡
Auto-Sync
potto sync appends missing keys with blank values. Never touches existing entries.
🔒
Secrets-Safe
Only reads key names, never exposes values. No risk of leaking secrets in logs or output.
🚦
CI-Friendly Exit Codes
Exit 0 = in sync. Exit 1 = out of sync. Gate your pipeline without extra scripting.
📂
Compare Any Two Files
potto compare .env.staging .env.production — works on any two env files, not just the defaults.
Usage

Three commands. That's it.

potto has a minimal, focused API that fits into any workflow.

$ potto   # same as `potto check`

-> Checking .env against .env.example

FAIL  2 key(s) in .env are MISSING from .env.example  (teammates can't run the app)
  - STRIPE_SECRET_KEY
  - REDIS_URL

WARN  1 key(s) in .env.example are MISSING from .env  (you may need to set these)
  - FEATURE_FLAG_NEW_UI

Run `potto sync` to fix .env.example automatically.
$ potto sync

-> Adding 2 key(s) to .env.example
  + STRIPE_SECRET_KEY=
  + REDIS_URL=

OK  2 key(s) added to .env.example

$ potto
OK  All 12 key(s) are in sync.
$ potto compare .env.staging .env.production

Comparing .env.staging vs .env.production

+  1 key(s) only in .env.staging:
  + STAGING_DEBUG_MODE

+  2 key(s) only in .env.production:
  + CDN_URL
  + SENTRY_DSN
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  env-sync:
    name: Check .env sync
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install potto
        run: cargo install potto
      - name: Check .env.example is up to date
        run: |
          cp .env.example .env
          potto check

# .git/hooks/pre-commit
#!/bin/sh
potto check || {
  echo "Commit blocked: run 'potto sync' to fix."
  exit 1
}

Exit codes

Code Meaning
0 Files are in sync
1 Files are out of sync
2 File read/write error
